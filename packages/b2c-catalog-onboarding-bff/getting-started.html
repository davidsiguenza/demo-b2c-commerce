B2C Catalog Uploader — Get started

A tiny BFF that turns a CSV or site-archive ZIP into a B2C Commerce catalog import — no Business Manager UI gymnastics. This page walks you through running it locally for the first time.

Tip: you can read this file by double-clicking getting-started.html in the project folder. No server needed yet.

1 Prerequisites

Install once on your machine:

  • Node.js 18+ — download from nodejs.org. Verify with:
node -v
  • pnpm — the package manager this project uses. Install with:
npm i -g pnpm

Plus access to a Salesforce B2C Commerce sandbox: a Business Manager user, an Account Manager API client, and a WebDAV password.

2 Get the code & configure .env

From a terminal:

git clone https://github.com/davidsiguenza/sfn-marketplace-bff.git cd sfn-marketplace-bff cp .env.example .env

Open .env and fill in every variable. Where each value lives:

Variable What it is Where to find it

3 Connect to the sandbox (one-time)

  1. In Account Manager (account.demandware.com) → API Client, create a client with scope SALESFORCE_COMMERCE_API:<tenant>. Save the client id and secret into .env.
  2. In Business Manager → Administration → Operations → Jobs:
    • Confirm the import job id is sfcc-site-archive-import (default in most BMs).
    • Create (if missing) a SearchReindex job for the storefront site, single step of type SearchReindex with "Product Search Index" + "Content Search Index" selected. Note the job id and paste it into REINDEX_JOB_ID if you want near-instant reindex.
  3. In BM → Administration → Organization → Users, click your user, set a WebDAV Access password. That's the value of WEBDAV_PASSWORD.

4 Install & run

pnpm install pnpm dev

The server boots on http://localhost:3001. Open it in your browser — you should see the admin UI. The ⓘ button in the header opens the in-app guide and a one-click self-test that hits every diagnostic endpoint.

First sanity check: open http://localhost:3001/diag/auth and http://localhost:3001/diag/data-api. Both should return { "ok": true, ... }. If either fails, jump to the FAQ below.

5 Upload your first catalog

On the home page, drop a .csv or .zip site-archive, pick (or type) a master catalog id, confirm the storefront catalog / pricebook / inventory list, and hit Upload. The BFF will preview, you confirm, and the import job runs in the background.

Need a starter? Click 📥 CSV template at the top right of the upload card.

It’s not working — FAQ

“pnpm: command not found”

You skipped step 1. Run npm i -g pnpm and try again.

/diag/auth returns 401 invalid_grant

The Account Manager client id/secret are wrong, or the client is missing the SALESFORCE_COMMERCE_API:<tenant> scope. Re-check the client in account.demandware.com.

/diag/data-api returns 401 / 403

Your AM client lacks Data API access on the sandbox. Add the Data API resource access in BM → Administration → Site Development → Open Commerce API Settings (Data tab) and grant read_write on /catalogs, /products, /price_books, /inventory_lists.

WebDAV upload fails with 401

Confirm the per-user WebDAV password is set in BM (the regular BM login password is not enough). Go to BM → Administration → Organization → Users → your user → WebDAV Access.

“JobNotFound” when triggering the import

The default import job id is sfcc-site-archive-import. Open BM → Administration → Operations → Jobs and confirm it exists with that exact id.

Products imported but the storefront still shows nothing

SFCC reindexes on a schedule (5–15 min). Set REINDEX_JOB_ID in .env to the SearchReindex job id from BM, restart the server, and the next upload will trigger it for you.

Live price edits return 404

OCAPI Data does not expose price_book_entries on every sandbox. Use the regular CSV/ZIP upload path — that always works.

Need more? The ⓘ button in the running admin shows the same guide plus a live self-test of /health, /diag/auth, /diag/data-api and /diag/edit-targets.
